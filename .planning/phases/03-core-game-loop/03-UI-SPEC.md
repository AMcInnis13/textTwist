## Phase 3 CSS Additions Required

Phase 3 must add one new CSS class to `styles.css`. The existing `.guess-correct` class (Phase 2 stub) must be **renamed** to `.guess-valid` and **expanded** with a background fill.

### Naming Discrepancy — Action Required

CONTEXT.md D-13 specifies the class name `.guess-valid`. `styles.css` currently defines `.guess-correct` (border-only, no background). Resolution:

1. Rename `.guess-correct` to `.guess-valid` in `styles.css`.
2. Add `-fx-background-color` to both feedback classes (see below).

### New / Corrected CSS to Write

```css
/* Section 9 replacement — guess feedback states */
.guess-valid {
    /* Valid word entered — green flash on guessDisplay HBox */
    -fx-background-color: #1b5e20;   /* dark green fill */
    -fx-border-color: #43a047;
    -fx-border-width: 2;
}

.guess-invalid {

    -fx-border-width: 2;
}
```

Both fills use dark variants of the feedback color so letter tile text (`#1a1a2e` on gold `.letter-slot`) remains legible during the flash. The border provides a secondary cue.

Source: `styles.css` existing padding declarations; `primary.fxml` hgap/vgap.

Exceptions:
- Letter button padding: `8 16 8 16` (vertical 8, horizontal 16) — asymmetric for readability; both values are multiples of 4.

## Component Inventory

### 1. Letter Buttons (dynamically created)

- **Node type:** `Button`
- **CSS class:** `.letter-btn` (added in `buildLetterButtons()`)
- **Created by:** `buildLetterButtons()` — called from `startGame()` and Twist handler
- **Container:** `letterButtonArea` (FlowPane, hgap=8, vgap=8)
- **Count:** Equals letter count for episode (3 at Level 1)
- **Text:** Single uppercase character (one letter of the target word)
- **Tracking:** By Button object reference in `letterButtonArea.getChildren()` — NOT by letter character (CLAUDE.md rule §4)

**States:**

| State | CSS Class | Visual |
|-------|-----------|--------|
| Enabled (unselected) | `.letter-btn` | Gold `#f0a500`, dark text, opacity 1.0 |
| Hover | `.letter-btn:hover` | Lighter gold `#ffc040` |
| Disabled (selected/used) | `.letter-btn:disabled` | Gray `#555555`, opacity 0.4 |

**State transitions:**
- `ENABLED → DISABLED`: user clicks the button (letter added to guess)
- `DISABLED → ENABLED`: Clear pressed, or Twist pressed (buttons recreated), or PauseTransition completes after Enter (both valid and invalid paths)

### 2. guessDisplay (HBox — fx:id locked)

- **Node type:** `HBox`
- **Permanent CSS class:** `.guess-display` (applied in FXML — never removed)
- **Temporary CSS classes:** `.guess-valid` or `.guess-invalid` (added/removed on Enter)
- **Children:** `Label` nodes with `.letter-slot` class, one per selected letter

**States:**

| State | CSS Classes on HBox | Visual |
|-------|---------------------|--------|
| Empty (idle) | `.guess-display` | Dark panel `#16213e`, centered, min-height 50px |
| Building (letters selected) | `.guess-display` | Same — children grow the row left-to-right |
| Valid word flash | `.guess-display .guess-valid` | Dark green fill `#1b5e20`, green border `#43a047` 2px |
| Invalid/duplicate flash | `.guess-display .guess-invalid` | Dark red fill `#b71c1c`, red border `#e53935` 2px |

**Class mutation rule (CLAUDE.md §6):**
```java
// Adding feedback class:
guessDisplay.getStyleClass().add("guess-valid");     // or "guess-invalid"

// Removing feedback class (in PauseTransition onFinished):
guessDisplay.getStyleClass().remove("guess-valid");  // or "guess-invalid"
```
Never use `removeAll()` on guessDisplay — it would remove `.guess-display` too.
Use `remove(String)` for the specific feedback class only.

### 4. timeLabel (Label — fx:id locked)

- **Permanent CSS classes:** `.info-label .time-label` (applied in FXML — never removed)
- **Temporary CSS classes:** `.timer-warning` or `.timer-critical` (added/removed by Timeline handler)

**States:**

| Seconds Remaining | CSS Classes | Text Color |
|-------------------|-------------|-----------|
| > 20s | `.info-label .time-label` | `#e0e0e0` (white) |
| ≤ 20s | `.info-label .time-label .timer-warning` | `#f0a500` (amber) |
| ≤ 10s | `.info-label .time-label .timer-critical` | `#e53935` (red) |

**Class mutation rule (CLAUDE.md §5 pitfall — CSS class accumulation):**
```java
// Every second in the Timeline KeyFrame handler:
timeLabel.getStyleClass().removeAll("timer-warning", "timer-critical");
if (secondsLeft <= 10) {
    timeLabel.getStyleClass().add("timer-critical");
} else if (secondsLeft <= 20) {
    timeLabel.getStyleClass().add("timer-warning");
}
// Above 20s: removeAll already cleared both — nothing more to add
```
`removeAll()` is correct here because `.info-label` and `.time-label` are NOT in the removal set. Only the dynamic classes are targeted.

Source: CONTEXT.md D-16; CLAUDE.md CSS accumulation pitfall.

## Interaction Contracts

### A. Letter Button Click

**Trigger:** User clicks an enabled letter Button.

**Sequence (all on FX thread — safe because JavaFX Timeline fires on FX thread):**
1. Disable the clicked Button: `btn.setDisable(true)`
2. Add Button to `pressedButtons` list: `pressedButtons.add(btn)`
3. Create new Label: `Label tile = new Label(btn.getText())`
4. Add CSS class: `tile.getStyleClass().add("letter-slot")`
5. Append to guess display: `guessDisplay.getChildren().add(tile)`

**Guard:** If `!playing` (episode ended), do nothing. Do not fire handler.

Source: CONTEXT.md D-01, D-02, D-03.

---

### B. Clear Button Click

**Trigger:** User clicks clearBtn.

**Sequence:**
1. For each node in `letterButtonArea.getChildren()`: cast to Button, call `setDisable(false)`
2. Clear `pressedButtons`: `pressedButtons.clear()`
3. Clear `guessDisplay.getChildren()`: `guessDisplay.getChildren().clear()`

**Guard:** If `!playing`, do nothing.

**No feedback flash** — immediate clear, no PauseTransition.

Source: CONTEXT.md D-06.

---

### C. Twist Button Click

**Trigger:** User clicks twistBtn.

**Sequence:**
1. Clear `pressedButtons`: `pressedButtons.clear()`
2. Clear `guessDisplay.getChildren()`: `guessDisplay.getChildren().clear()`
3. Clear `letterButtonArea.getChildren()`: `letterButtonArea.getChildren().clear()`
4. Call `buildLetterButtons()` — creates new Button objects in new shuffled order
5. Disable `lastWordBtn`: `lastWordBtn.setDisable(true)`

**Guard:** If `!playing`, do nothing.

**No feedback flash** — immediate reshuffled layout.

Note: Existing letter Button object references in `lastWordButtons` become stale after Twist (those nodes are removed from the scene). Disabling `lastWordBtn` (step 5) prevents Last Word from referencing them.

Source: CONTEXT.md D-07, D-08.

---

### D. Enter Button Click

**Trigger:** User clicks enterBtn.

**Sequence — common path:**
1. Derive guess string: `pressedButtons.stream().map(Button::getText).collect(Collectors.joining())`
2. If `pressedButtons` is empty: do nothing (no feedback). Return.
3. Call `twistController.checkGuessWord(guessWord)` → `int points`

**Sequence — valid word (points > 0):**
4. Add word to found list: `foundWords.add(guessWord)` (updates ListView immediately)
5. Update score: `scoreLabel.setText("Score: " + twistController.getScore())`
6. Copy `pressedButtons` to `lastWordButtons`: `lastWordButtons = new ArrayList<>(pressedButtons)`
7. Enable Last Word button: `lastWordBtn.setDisable(false)`
8. Check target word: if `guessWord.equalsIgnoreCase(twistController.getTargetWord())`, call `endEpisode()` and return immediately — skip the flash sequence.
9. Add feedback class: `guessDisplay.getStyleClass().add("guess-valid")`
10. Start `PauseTransition(Duration.millis(500))`; in `onFinished`:
    - `guessDisplay.getStyleClass().remove("guess-valid")`
    - Clear guess: clear `pressedButtons`, clear `guessDisplay.getChildren()`
    - Re-enable all letter buttons (same as Clear sequence)

**Sequence — invalid or duplicate word (points == 0):**
4. Add feedback class: `guessDisplay.getStyleClass().add("guess-invalid")`
5. Start `PauseTransition(Duration.millis(500))`; in `onFinished`:
    - `guessDisplay.getStyleClass().remove("guess-invalid")`
    - Clear guess: clear `pressedButtons`, clear `guessDisplay.getChildren()`
    - Re-enable all letter buttons

**Guard:** If `!playing`, do nothing.

**Score update timing:** Score label updates BEFORE the PauseTransition (step 5 in valid path) — player sees new score during the green flash.

**Target word check timing:** Target word check (step 8) occurs BEFORE the PauseTransition starts. If target word matched, `endEpisode()` is called immediately with no flash delay.

Source: CONTEXT.md D-11, D-12, D-13, D-18, D-23.

---

### E. Last Word Button Click

**Trigger:** User clicks lastWordBtn (only possible when enabled — after first valid word, before next Twist).

**Sequence:**
1. Copy `lastWordButtons` into `pressedButtons`: `pressedButtons = new ArrayList<>(lastWordButtons)`
2. For each Button in `pressedButtons`: `btn.setDisable(true)` (mark buttons as used)
3. Clear `guessDisplay.getChildren()`
4. For each Button in `pressedButtons`: create `Label(btn.getText())`, add `.letter-slot`, append to `guessDisplay.getChildren()`

**Guard:** Button is disabled until first valid word — no explicit guard needed in handler body.

**No feedback flash** — immediate reconstruction of last guess state.

Source: CONTEXT.md D-09, D-10.

---

### F. Timeline Tick (every 1 second)

**Trigger:** `javafx.animation.Timeline` KeyFrame at `Duration.seconds(1)` interval, `INDEFINITE` cycle count.

**Sequence:**
1. Decrement `secondsLeft` by 1
2. Update timer display: `timeLabel.setText(String.format("%d:%02d", secondsLeft / 60, secondsLeft % 60))`
3. Update timer CSS:
   ```
   timeLabel.getStyleClass().removeAll("timer-warning", "timer-critical");
   if (secondsLeft <= 10) add "timer-critical"
   else if (secondsLeft <= 20) add "timer-warning"
   ```
4. If `secondsLeft <= 0`: call `endEpisode()` (which stops the Timeline)

**Initial display:** `timeLabel.setText("2:00")` — set before Timeline starts in `startGame()`.

**Thread safety:** Timeline fires on the FX thread — direct label updates are safe without `Platform.runLater()`.

Source: CONTEXT.md D-14, D-15, D-16, D-17; CLAUDE.md rules §1, §6.

---

### G. endEpisode() — Single Entry Point

**Callers:** Timer expiry (Timeline tick, step 4) and target word guessed (Enter handler, step 8). Both paths must route through this method (CLAUDE.md rule §5).

**Sequence in Phase 3 (minimal — Phase 4 expands):**
1. Set `playing = false` (guards all button handlers)
2. Stop the Timeline: `gameTimeline.stop()`
3. Disable all interactive controls: `twistBtn`, `enterBtn`, `lastWordBtn`, `clearBtn`, all letter buttons
4. (Phase 4 will add: evaluate levelScore vs threshold, show overlay, advance or game-over)

**Phase 3 scope:** `endEpisode()` stops the game and disables UI. It does NOT advance levels, show dialogs, or evaluate the score threshold — those are Phase 4.

Source: CONTEXT.md D-17, D-18; CLAUDE.md rule §5.

## Initialization Sequence

Called by FXMLLoader on scene load. Must complete before any user interaction.

```
initialize()
  ├── foundWords = FXCollections.observableArrayList()
  ├── foundWordsList.setItems(foundWords)
  └── startGame()
        ├── twistController = new TwistController("twister_words")
        ├── twistController.beginEpisode(3)   ← Level 1: 3-letter words
        ├── playing = true
        ├── secondsLeft = 120
        ├── timeLabel.setText("2:00")
        ├── buildLetterButtons()
        │     ├── letterButtonArea.getChildren().clear()
        │     ├── List<String> letters = twistController.shuffleLetters()
        │     └── for each letter: new Button(letter), add ".letter-btn", wire handler, add to letterButtonArea
        └── gameTimeline = new Timeline(new KeyFrame(Duration.seconds(1), e -> onTick()))
              └── gameTimeline.setCycleCount(Timeline.INDEFINITE)
              └── gameTimeline.play()
```

## State Machine

Phase 3 uses a `boolean playing` field. Phase 4 will expand to a `GameState` enum.

| State | `playing` | UI Behavior |
|-------|-----------|-------------|
| PLAYING | `true` | All button handlers active; Timeline running |
| ENDED | `false` | All handlers silently return; controls disabled; Timeline stopped |

Transition: `PLAYING → ENDED` via `endEpisode()` only. No transition back in Phase 3.

Source: CONTEXT.md D-22.

## Edge Cases and Guard Contracts

| Scenario | Handling |
|----------|---------|
| User clicks Enter with empty `pressedButtons` | Return immediately — no feedback flash, no score change |
| User submits duplicate word | `checkGuessWord` returns 0; treated as invalid; `.guess-invalid` flash fires |
| Duplicate letters in target word | Letter tracking is by Button object reference (not character) — each Button is unique; no special case needed |
| User clicks letter button after `endEpisode()` | `!playing` guard returns immediately |
| User clicks control button after `endEpisode()` | Controls disabled in `endEpisode()` — handler never fires |
| `lastWordButtons` references after Twist | `lastWordBtn` disabled in Twist handler (D-08); stale references are never accessed |
| PauseTransition `onFinished` fires after `endEpisode()` | Harmless — clearing children of a disabled scene is idempotent; `playing` check not needed in `onFinished` body |
| Timer reaches exactly 0 | `secondsLeft <= 0` check fires `endEpisode()`; Timeline stopped; no double-call risk if guard is set before stop |
